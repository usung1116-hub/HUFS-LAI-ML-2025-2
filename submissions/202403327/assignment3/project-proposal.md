# Project Proposal

## 해결하고자 하는 문제
토익 공부 중 철자는 같지만 품사와 문맥에 따라 뜻이 완전히 달라지는 **'다의어'** 때문에 해석에 어려움을 겪습니다. 특히 **`address`, `account`, `present`, `contract`, `appreciate`** 이 5개 빈출 단어는 명사/동사 여부에 따라 의미가 극단적으로 달라져(예: `contract` - '계약서' vs '병에 걸리다'), 실전 독해 시 문맥 파악에 많은 시간을 뺏깁니다.

**실제 사용 시나리오:** 토익 오답을 공부할 때, 위 5개 단어가 포함된 문장이 나올 때마다 이 도구를 사용해 정확한 의미를 검증하고 학습합니다. 만든 후 최소 5회 이상 꾸준히 활용할 계획입니다.

## 제안하는 솔루션
**토익 빈출 다의어 5종 판별기**
토익 빈출 혼동 어휘 5개(`address`, `account`, `present`, `contract`, `appreciate`)에 특화된 문맥 기반 의미 분류기입니다.

**주요 기능:**
* **타겟 단어 인식:** 입력 문장에 5개 단어 중 하나가 있는지 자동 확인
* **의미 및 품사 분류:** 문맥을 분석하여 해당 단어의 정확한 뜻 판별
    * 예: `contract` + `virus` -> **'병에 걸리다(Verb)'**
    * 예: `sign` + `contract` -> **'계약(Noun)'**
* **결과 출력:** 예측된 의미와 확률 표시

## ML 활용 계획

**Assignment 4:**
* **데이터 수집:** 네이버 영어사전 예문 크롤링 (단어당 40개 정도)
* **데이터 증강:** `contract(동사)`와 같이 사전에서 찾기 힘든 희소 클래스는 ChatGPT 등을 활용하여 토익 난이도의 합성 데이터를 생성해 클래스 불균형 해소
* **라벨링:** 수집된 문장에 대해 Class ID 부여

**Assignment 5:**
* **모델 학습:** 문맥 기반 다중 분류 모델 학습 (Naive Bayes 또는 Logistic Regression)
* **특징 추출:** TF-IDF Vectorizer 및 Lemmatization(동사 원형 복원) 활용
* **평가:** Test set에 대한 Accuracy 측정 및 Confusion Matrix 분석

**Assignment 6:**
* 실제 토익 파트 5 문제나 고난도 예문을 모델에 입력하여 성능 검증
* 잘못 분류된 케이스 분석 및 최종 보고서 작성
