# 프로젝트 명: 투수의 다음 투구 예측 봇

## 1. 해결하고자 하는 문제
야구 경기를 볼 때 투수와 타자의 수싸움이 치열한 중요한 순간에서 투수가 다음 공으로 무엇을 던질까라는 
궁금증이 항상 있었습니다
하지만 저의 얕은 지식으로는 알 수 없고 해설자 또한 공을 맞추어내기가 쉽지 않아보였습니다.
그래서 데이터에 기반한 예측이 좀 더 정확할 것이라고 생각했습니다.
또한 어렸을 때 야구 팀 안에서 데이터 기반 예측가로 한번 활동해보고 싶었던 기억이 생각나 
오늘날 이 꿈을 약간이라도 실현할수 있는 주제로 선택해보았습니다.

## 2. 제안하는 솔루션

### 1. 어떤 서비스를 만들 것인지
1차 목표는 local에서 실행하는 python script 또는 간단한 web application(아마 제미나이 추천인 streamlit)입니다.
사용자가 특정 투수를 선택하고 현재 경기 상황을 입력하면 다음 투구 구종별 확률을 보여주는 형식으로 제공될 예정입니다

### 2. 주요 기능
*기능 1*: **현재 상황 입력**: 사용자가 *투수*, *타자*, *볼타운트*, *아웃 카운트*, *주자 상황* 등을 입력

*기능 2*: **다음 투구 예측**: '예측하기' 버튼을 누르면 사전에 학습된 ML모델이 입력된 상황을 바탕으로 다음 투구 구종의 확률을
계산하여 보여줌

## 3. 실제 사용 시나리오

1. 야구 경기를 시청
2. 중요한 순간에 만들어둔 프로그램을 엶
3. 현재 투수를 선택하고 현재 볼카운트, 아웃, 주자 상황을 입력
4. '예측' 버튼을 클릭
5. "직구:50%, 체인지업:20%, 커브:30%"와 같은 결과를 확인하고 실제 투구와 비교

## 4. ML 활용 계획

### 1. 데이터 수집 방법(assignment4)
알아보니 KBO 데이터는 pitch-by-pitch 데이터 확보가 어렵다고 해서 MLB 데이터를 사용할 것 같습니다
pybaseball 같은 파이썬 라이브러리를 사용하면 MLB의 Statcast 데이터를 pitch-by-pitch 단위로 쉽게 수집할 수 있다고 합니다.

*수집할 feature*:이전 투구 구종, 이전 투구 결과(스트라이크/볼/파울/안타), 볼카운트, 아웃 카운트, 등등 ..

*수집할 label*: 현재 투구 구종

### 2. 어떤 모델을 학습시킬지(assignment 5)

이 문제는 다음구종이 무엇인지를 맞추는 다중 클래스 분류 문제이기 때문에 다음과 같은 모델을 학습시킬 예정입니다.

*시도할 모델*: Decision Tree/Random Forest 또는 XGBoost/LightGBM

*학습 계획*: 특정 투수의 2023-2025(연도는 바뀔 수 있음)시즌 데이터를 모두 수집한 뒤, 80%는 학습에 20%는 검증에
사용하여 모델의 정확도를 평가할 예정
