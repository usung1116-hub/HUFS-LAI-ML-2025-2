# 📧 프로젝트 제안서: 교수님 안심 전송기 (Politeness Check Bot)

## 1. 해결하고자 하는 문제

###  본인이 겪고 있는 실제 문제가 무엇인가?
* 교수님이나 조교님, 혹은 학과 사무실에 문의 메일/문자를 보낼 때마다 **내 말투가 예의에 어긋나지 않는지 과도하게 걱정**합니다.
* "~인데요"라고 써도 되는지, 무조건 "~입니다"로 끝내야 하는지 고민하느라 메일 작성에 **불필요하게 긴 시간(10분 이상)을 소모**합니다.
* 작성 후에도 "혹시 기분 나쁘게 들리면 어떡하지?"라는 불안감에 전송 버튼을 누르기 힘들어합니다.

###  왜 이 문제를 해결하고 싶은가?
* 메일 작성에 드는 불필요한 **'감정 소모'와 '시간'을 줄이고 싶습니다.**
* 언어공학 전공생으로서, 기계학습을 통해 **'공적인 언어'와 '사적인 언어'의 차이를 정량적으로 구분**해보고 싶습니다.

###  이 도구를 본인이 실제로 사용할 것인가?
* **네.** 학기 중 교수님께 과제 제출 메일을 보내거나 출결 문의를 할 때 **반드시 사전에 검사하는 용도**로 사용할 것입니다. (한 학기에 최소 10회 이상 사용 예상)


## 2. 제안하는 솔루션

###  어떤 도구/서비스를 만들 것인가?
* 사용자가 작성한 문장을 입력하면, 그 문장의 **'정중함 점수(Politeness Score)'**를 판단해주는 **텍스트 분류기(Text Classifier)**입니다.
* 내 문장이 **'격식체(Formal)'**인지 **'비격식체(Informal)'**인지 판단하여 결과를 알려줍니다.

###  주요 기능은 무엇인가?
1. **텍스트 입력 및 분석:** 작성한 메일 본문을 입력받습니다.
2. **정중함 확률 출력:** _"이 문장은 85% 확률로 정중합니다"_ 또는 _"이 문장은 90% 확률로 친구에게 하는 말 같습니다"_와 같은 피드백을 제공합니다.
3. **안심 피드백:** 특정 임계값(예: 70점)을 넘으면 **"전송해도 좋습니다!"**라는 메시지를 띄워 심리적 안정을 줍니다.

###  사용자(본인)가 어떻게 사용하게 될 것인가?
1. 메일이나 메신저에 보낼 내용을 평소처럼 작성합니다.
2. 전송 전, 이 텍스트를 복사하여 프로그램(터미널 혹은 웹)에 붙여넣습니다.
3. 결과가 **'부적절(Informal)'**로 나오면 문장을 수정한 뒤 다시 검사합니다.
4. **'적절(Formal)'** 판정을 받으면 안심하고 실제 메일을 전송합니다.


## 3. ML 활용 계획

###  Assignment 5에서 어떤 모델을 학습시켜볼 계획인가?
* 텍스트 데이터를 입력받아 `0(비격식/반말)`과 `1(격식/존댓말)`을 구분하는 **이진 분류(Binary Classification) 모델**을 직접 학습시킬 계획입니다.

###  어떤 데이터를 수집할 예정인가? (Data Collection)
| Class | Label | 데이터 소스 | 수량 (예상) |
|:---:|:---:|:---|:---:|
| **Class 0** | 비격식 | 친한 친구들과 나눈 **카카오톡 대화** (내보내기 기능 활용) | 50~100 문장 |
| **Class 1** | 격식 | 학교 홈페이지 공지사항, 뉴스 기사, 과거 교수님께 보낸 **정중한 메일** | 50~100 문장 |

* 위 데이터를 엑셀(`.csv`)이나 텍스트 파일(`.txt`)로 정리하여 **Training Set**을 구축할 것입니다.

###  (선택) 어떤 ML 기법/모델을 생각 중인가?
* **Feature Extraction:** 데이터가 텍스트이므로 문자를 숫자로 바꾸기 위해 **TF-IDF (Term Frequency-Inverse Document Frequency)** 벡터화 방식을 사용할 예정입니다.
* **Model:** 적은 데이터로도 학습이 잘 되고 텍스트 분류의 기초가 되는 **Naive Bayes** 혹은 **Logistic Regression** 모델을 `scikit-learn` 라이브러리를 통해 구현할 것입니다.
