# 3차 과제 제안서 (202402960 신새봄)

## 스페인어 동사 분류기

### 필요성
- 저의 전공언어인 스페인어는 동사의 변화가 다양하고 복잡한 언어입니다. 같은 동사라도 법과 시제, 인칭에 따라 100가지 이상의 변화형을 가지고 있습니다. 그래서 공부를 하다보면 문장 속에서 변형된 동사를 보았을 때 '이 동사가 어떤 법의 어떤 시제이지?'하고 헷갈리는 순간이 자주 있습니다.
- 하지만 기존의 사전이나 번역기는 동사의 문법적 역할보다는 의미에 초점을 두어 문법적인 형태 정보를 제공하지 않습니다. 따라서 저는 이번 프로젝트에서 스페인어 학습 시 유용하게 이용할 수 있는 스페인어 동사 분류기를 만들고자 합니다.

### 솔루션
- 변형된 형태의 스페인어 동사를 입력하면, 그 동사의 문법적 형태 정보를 자동으로 분석해주는 도구를 만들 예정입니다.
- 예) [habláremos] -> [접속법, 미래, 1인칭, 복수]
- 주요 기능
  - 동사의 철자만으로 법(Mood), 시제(Tense), 인칭(Person), 수(Number)를 자동 분류함
  - 규칙/불규칙 동사 모두 지원함
- 저는 이 모델을 스페인어 읽기 수업이나 문법 설명이 부족한 교재를 공부할 때 사용할 것입니다.

### ML 활용 계획
- 데이터 수집 및 분석
  - [unimorph spanish] (https://github.com/unimorph/spa/blob/master/spa.derivations)
  - 우선, 규칙 기반 활용형 생성기를 만들고 위의 데이터를 활용하여 [동사-라벨]의 데이터셋을 만들 예정입니다. (학습 가능한 데이터 양을 늘리기 위함)
  - 예) {"verb" : "habláremos", "mood" : "subjunctive", "tense" : "future", "person" : 1pl}
  - 데이터셋을 만든 이후에는 클래스별 분포를 확인하여 만약 데이터가 부족하거나 불균형하다면 보완할 다른 데이터를 수집하여 추가할 예정입니다.

- 모델 학습 및 평가
  - 동사 분류기를 만들기 위해 저는 Char 기반의 Multi-Head LSTM 분류 모델을 학습시킬 예정입니다. input이 문장이 아닌 단어이기 때문에 문맥의 고려가 필요하지 않다고 생각하여 Char기반의 모델을 선택하였습니다. 
    - 동사를 문자 단위로 임베딩하여 LSTM으로 특징을 추출한 후 법,시제,인칭 등을 각각 동시에 예측하는 구조로 만들 것입니다.
  - 평가는 Accuracy, Macro-F1등을 통해 성능을 확인할 예정입니다.
