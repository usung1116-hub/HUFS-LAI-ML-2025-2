# '일일 뉴스 자동 분류' 및 '카테고리별 핫 토픽' 추출기

## 1. 해결하고자 하는 문제

- 저는 매일 아침 **영어 학습 및 시사 교양 습득**을 목표로 영어 뉴스를 읽습니다. 하지만 아침의 한정된 시간 동안 수많은 기사가 쏟아져 나오는 상황에서, 모든 분야(정치, 사회, 경제, IT 등)가 **혼재된 뉴스 피드**를 일일이 확인하는 것은 매우 비효율적입니다. 이로 인해 제가 정말 관심 있는 분야의 핵심 이슈를 신속하게 파악하는 데 어려움을 겪고 있습니다. 대신, **'오늘 [IT] 분야에서 가장 중요한 주제'** 또는 **'오늘 [경제] 분야의 핵심 이슈'** 가 무엇인지 **'분야별로 자동 분류되고 정제된'** 형태로 한눈에 빠르게 파악하여, 아침 시간을 효율적으로 활용하고자 합니다.

## 2. 실제 사용 시나리오 

- **매일 자정**, 스크립트를 실행
- 스크립트가 오늘 하루 동안의 뉴스 헤드라인 수백 개를 수집
- 학습된 ML 모델이 이 헤드라인들을 '정치', '경제', 'IT/과학', '사회', '스포츠' 등 **내가 정의한 카테고리**로 자동 분류    
- **결과:** "오늘 \[IT/과학] 카테고리 뉴스는 50개였으며, 이 중 핵심 키워드는 'AI 반도체', 'NVIDI'입니다." 와 같은 요약 리포트


## 3. 제안하는 솔루션 

- 네이버 뉴스 등에서 특정 카테고리(예: IT/과학)의 과거 뉴스 헤드라인을 크롤링하여 **학습 데이터(X)** 를 구축, 해당 지정된 카테고리(예: 'IT/과학')를 **라벨(Y)** 로 사용합니다.
- '헤드라인 텍스트(X)'를 입력받아 '뉴스 카테고리(Y)'를 예측하는 **텍스트 분류(Text Classification) ML 모델**을 구축합니다.
- 매일 새 뉴스를 수집해 이 모델로 자동 분류하고, 분류된 그룹 내에서 키워드 빈도를 분석하여 '카테고리별 핫 토픽'을 선정하여 요약 리포트를 작성합니다.


## 4. ML 활용 계획

- **Assignment 4: (데이터 수집 및 라벨링)**
    - 네이버 뉴스의 각 카테고리(정치, 경제, IT/과학, 사회, 스포츠)별로 과거 뉴스 헤드라인을 크롤링합니다. 
    - **Features (X):** '헤드라인 텍스트'
    - **Label (Y):** '카테고리명' (예: '정치', '경제', 'IT/과학').
    
- **Assignment 5: (모델 학습 및 평가)**
    
    - **목표:** MLPClassifier를 학습 데이터를 이용해 학습시키고, 비교군으로 Logistic Regression 모델도 함께 학습시켜 MLP와 성능을 비교하고자 합니다.
            
    - **평가:** Train/Test 셋을 분리하여 **F1-Score**  및 **정확도(Accuracy)** 를 측정합니다.
        
- **Assignment 6: (실제 사용 및 개선)**
    
    - Assignment 5에서 학습된 최적의 분류 모델(예: MLPClassifier)을 실제 스크립트에 탑재합니다.
    - 매일 자정, **카테고리 구분이 없이** 오늘의 뉴스를 크롤링합니다.
    - 가져온 새 헤드라인들을 **모델에 입력하여** '정치', '경제', 'IT' 등으로 **자동 분류**합니다.
    - **(핫 토픽 추천):** 분류된 각 카테고리 그룹(예: 'IT'로 분류된 50개 헤드라인) 내에서만 **키워드 빈도수**를 계산하여, "오늘 [IT] 분야 핫 토픽: 'AI 반도체'"와 같이 카테고리별 핫 토픽을 추출해 보여줍니다.
