# 학교 홈페이지 날짜별 정리 도구

## 해결하고자 하는 문제
내가 학교 행사나 중요한 공지사항을 확인하기 위해서는 크게 외대 메인 홈페이지, 본전공 홈페이지, 이중전공 홈페이지 등의 여러 페이지를 방문해야 한다. 그러나 매번 다른 사이트를 여러번 확인하기에는 시간이 낭비라고 느껴졌다. 또한 메인 홈페이지의 경우 HUFS공지, 학사공지, 장학공지를 매번 열어보며 확인하기 불편했다. 마감기한이나 구체적인 날짜도 섞여 있어 내게 급한 것 순서대로 확인하기도 불편했다. 메인 홈페이지와 학과 공지가 중복되는 내용도 있어서 저번에 본 공지인지 확인하기도 불편하다. 나에게 필요한 사항을 중복되지 않게 내게 필요한 것만 시간 순으로 알려주는 모델을 만들어 해결하겠다.

## 해결 방법
- 사용 방법 및 목적
  - 입력: 학교 여러 홈페이지의 공지사항 내용들.
  - 출력: 나에게 관련 있는 내용들만 공지 날짜, 마감 날짜를 중복되는 알림은 제거하여 알려준다. 마감일이 있는 공지 같은 경우 중요도 순으로 알려준다.
  - 목적: 중복되지 않는 학교 전체 공지사항을 개인 맞춤형으로 날짜순으로 안내받아 중요한 공지를 놓치지 않게 하기 위해서이다.

## 세부 계획

### Assignment 4
- 데이터셋: 학교 메인 홈페이지 중 HUFS 공지, 장학 공지, 학사 공지, 본전공과 이중전공 홈페이지 등 나와 관련 있는 사이트의 데이터 크롤링.
- 학교 사이트에서 관련 공지사항을 크롤링한다.
- 공지사항의 내용을 전부 또는 일부 포함할 것이다 (아마 약 첫 512토큰 정도가 될 것 같지만 변경 가능하다)
- 제목과 게시 날짜, 마감 기한, 공지 유형 등을 라벨링한다.
- 중복되는 공지사항 쌍도 만든다. 제목이 달라도 내용이 같은 경우가 있기 때문이다.
- 나와 관련 있는지 여부와 나와의 중요도를 직접 라벨링한다.

### Assignment 5
- 모델: 아마 KoBERT를 사용할 것 같다(데이터셋 양이나 성능에 따라 변경 가능)
- Assignment4를 사용한 데이터셋을 학습시킨다.
- Evaluation
  -  나와의 적합성, 중요도 등을 정확히 판단했는지 복합적으로 알아본다. train, validation에 사용하지 않은 데이터셋으로 정합성과 중요도를 각각, 그리고 동시에도 고려해서 precision, recall, f1 score 등을 사용한다.

### Assignment 6
- 일정 기간을 두고 (또는 노션, 구글 캘린더 또는 다른 내가 자주 사용하는 다른 기능으로 생각중)으로 공지한 날짜와 마감일 순으로 정리해준다.
- 마감일이 있고, 같은 날짜인 경우 중요도 순으로 정리해서 알려준다.
- 마감 기간이 3일 이내로 남은 경우이거나 선착순인 경우 즉시 이메일로 알림을 보낸다.
- 또는 나만의 공지사항 확인용 탭을 시각화해서 여기로 알림을 전달하게 할 수도 있다.
