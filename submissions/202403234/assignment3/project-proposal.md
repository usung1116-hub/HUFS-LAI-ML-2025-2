# 일정 자동 분류 및 우선순위 리포트 생성기

## 1. 해결하고자 하는 문제
학기 중에는 과제, 시험, 발표, 동아리 활동 등 다양한 일정이 동시에 쏟아진다.
하지만 일정 관리 앱이나 메모장은 단순히 할일만 기록할 뿐 어떤 일정이 더 중요하고 시급한지 자동으로 구분해주지 못한다.
그래서 중요한 과제의 마감일을 놓치거나, 우선순위를 잘못 판단해 효율적으로 일정을 끝마치지 못한 경험이 많다.
따라서 입력된 일정 문장을 자동으로 분류하고,그 중요도를 예측해주는 시스템을 만들고자 한다.
매주 실제 일정 데이터를 입력해 결과를 확인하며,2주 이상 사용해 모델의 예측 정확도와 활용성을 검증할 계획이다.

## 2. 제안하는 솔루션
텍스트로 입력한 일정을 AI가 자동으로 분석하여
카테고리 분류(과제, 시험, 발표, 기타)와 우선순위(High/Medium/Low)를 예측하고,
주간 리포트 형태로 요약해주는 웹 서비스이다.일정 문장과 마감일, 키워드를 분석하여
자동으로 카테고리와 중요도를 예측하고, 우선순위에 따른 주간 요약표로 시각화한다.
사용자는 Gradio 대시보드에서 텍스트로 일정을 입력하고 결과를 확인할 수 있으며,
각 주의 일정 분포와 우선순위를 한눈에 볼 수 있다.

## 3. ML 활용 계획
직접 수집한 일정 데이터와 공개 일정 텍스트 데이터를 함께 사용하여
문장 기반 텍스트 분류 + 중요도(우선순위) 예측 모델을 학습시킬 계획이다.
데이터 구성은 다음과 같다.
직접 수집 데이터: 학기 중 실제 일정 약 50건(과제, 발표, 시험, 학생회 활동 등)을 입력하고
카테고리(과제/시험/발표/기타)와 중요도(High/Medium/Low)를 수동으로 라벨링.
보강 데이터: Kaggle과 GitHub의 공개 일정·할일(Todo) 분류 데이터셋 약 300~400건을 번역 및 가공하여 병합.
이를 통해 학습용 데이터의 다양성과 양을 확보하여 모델의 일반화를 높인다.
모델 학습 과정은 다음과 같다.
전처리: 일정 문장에서 불용어 제거, 날짜/시간 정규화, 마감일까지 남은 일수(days_left) 계산
특징 추출: TF-IDF 기반 텍스트 벡터화 후 days_left 및 keyword_score(긴급 단어 포함 여부)를 추가 feature로 사용
모델 구성: Logistic Regression과 RandomForestClassifier를 각각 학습시켜 성능 비교
평가 지표:
카테고리 분류 → Macro-F1 score
중요도 예측 → Accuracy 및 Confusion Matrix
확장 계획: 데이터가 누적되면 KoBERT를 활용해 한글 문장 분류 모델로 고도화
학습된 모델은 입력된 일정을 자동으로 분류하고
“이번 주 High 우선순위 일정 3건”과 같은 형태로 리포트를 생성한다.
결과는 막대그래프 형태로 시각화되어,
사용자가 주간 일정의 업무량과 우선순위를 직관적으로 파악할 수 있도록 설계할 예정이다.
